## 🧩 Topics Covered

### 1. 🔒 Encapsulation
Bundling data (fields) and methods that operate on the data into a single unit (class), and restricting direct access using access modifiers.

- Private fields with public getters/setters
- Data hiding and controlled access

---

### 2. 🧬 Inheritance
Mechanism where one class acquires the properties and behaviors of another class.

- Single Inheritance
- Multilevel Inheritance
- Hierarchical Inheritance
- `super` keyword usage

---

### 3. 🎭 Polymorphism
Ability of an object to take many forms.

- **Compile-time (Static):** Method Overloading
- **Runtime (Dynamic):** Method Overriding
- `@Override` annotation

---

### 4. 🫥 Abstraction
Hiding implementation details and showing only the essential features.

- Abstract classes and methods
- Real-world abstraction examples

---

### 5. 🔌 Interface
Blueprint of a class — defines what a class must do, not how.

- Implementing interfaces
- Multiple interface implementation
- Food problem (real-world interface use case)

---

### 6. 🏠 Inner Class
A class defined within another class.

- Static Nested Class
- Non-static Inner Class
- Anonymous Inner Class

---

### 7. 🏗️ Constructor
Special method called when an object is created.

- Default Constructor
- Parameterized Constructor
- Constructor Overloading
- Constructor Chaining using `this()` and `super()`

---

### 8. 🧪 Cloning
Creating an exact copy of an object.

- **Shallow Copy** — copies object references
- **Deep Copy** — copies actual nested objects

---

### 9. 🔗 Relationship & Behaviour
Modeling real-world relationships between objects.

- **Association** — one-to-one, one-to-many
- **Aggregation** — HAS-A (weak relationship)
- **Composition** — HAS-A (strong relationship)
